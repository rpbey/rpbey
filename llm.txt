# Material Design 3 (M3) - LLM Reference Guide

> Source: https://m3.material.io/
> Version: M3 Expressive (May 2025)

## Overview

Material Design 3 is Google's open-source design system for building beautiful, usable products.
M3 Expressive adds vibrant colors, intuitive motion, adaptive components, flexible typography, and contrasting shapes.

Key Features:
- Dynamic Color: Personalized, accessible color schemes
- Motion Physics: Spring-based animations
- Expressive Components: 15+ new/updated components
- Shape Library: 35 shapes with morphing
- Typography: 30 type styles (15 baseline + 15 emphasized)

---

## Foundations

### Design Tokens
Design tokens are reusable design decisions that make up a design system's visual style.

Token Structure:
- `md.ref.*` - Reference tokens (static values: colors, fonts, measurements)
- `md.sys.*` - System tokens (theme decisions, can change by context)
- `md.comp.*` - Component tokens (element-specific: container, label, icon)

Example: `md.sys.color.secondary-container` → `md.ref.palette.secondary90` → `#E8DEF8`

Contexts: light/dark theme, device form factors, dense layouts, RTL

### Accessibility
- Three contrast levels: Standard, Medium, High
- All color roles support accessible contrast
- User-controlled contrast via tokens

### Layout
- Adaptive design for multiple screen sizes
- Responsive breakpoints
- Density adjustments

### Interaction States
- Enabled, Disabled, Hover, Focus, Pressed, Dragged

## Styles

### Color System

26+ color roles mapped to Material Components:

**Primary**
- `primary` - Main brand color
- `on-primary` - Text/icons on primary
- `primary-container` - Container backgrounds
- `on-primary-container` - Text on primary container

**Secondary**
- `secondary`, `on-secondary`, `secondary-container`, `on-secondary-container`

**Tertiary**
- `tertiary`, `on-tertiary`, `tertiary-container`, `on-tertiary-container`

**Error**
- `error`, `on-error`, `error-container`, `on-error-container`

**Surface** (tone-based, not elevation-based)
- `surface`, `surface-dim`, `surface-bright`
- `surface-container-lowest`, `surface-container-low`
- `surface-container`, `surface-container-high`, `surface-container-highest`
- `on-surface`, `on-surface-variant`
- `outline`, `outline-variant`
- `inverse-surface`, `inverse-on-surface`, `inverse-primary`
- `scrim`, `shadow`

**Fixed Colors** (same in light/dark)
- `primary-fixed`, `primary-fixed-dim`, `on-primary-fixed`, `on-primary-fixed-variant`
- Same pattern for secondary/tertiary

Dynamic Color Features:
- User-generated color (from wallpaper)
- Content-based color (from images)
- Automatic dark theme generation

### Typography

30 type styles in 5 roles:

| Role | Sizes | Use |
|------|-------|-----|
| Display | Large, Medium, Small | Hero text, splash screens |
| Headline | Large, Medium, Small | Screen titles |
| Title | Large, Medium, Small | Section headers |
| Body | Large, Medium, Small | Paragraph text |
| Label | Large, Medium, Small | Buttons, chips, captions |

Each has baseline + emphasized variant (total 30 styles).

Typography Tokens:
- `md.sys.typescale.display-large`
- `md.sys.typescale.display-large-emphasized`
- Font, line height, size, tracking, weight

Default Font: Roboto Flex (variable font with axes: weight, width, grade, slant, optical size)

### Shape

Corner Radius Scale:
- None: 0dp
- Extra Small: 4dp
- Small: 8dp
- Medium: 12dp
- Large: 16dp
- Large Increased: 20dp
- Extra Large: 28dp
- Extra Large Increased: 32dp
- Extra Extra Large: 48dp
- Full: 50% (pill shape)

Shape Principles:
- Use abstract shapes sparingly
- Create tension with contrasting shapes (round + square)
- Shape morphing for interaction states
- Shapes are versatile, not semantic (no fixed meaning)

Shape Library: 35 decorative shapes with built-in morph animations

### Elevation

5 Levels (expressed as tonal color, not just shadows):
- Level 0: 0dp - Surface
- Level 1: 1dp - Low emphasis
- Level 2: 3dp - Cards, sheets
- Level 3: 6dp - FAB, Nav drawer
- Level 4: 8dp - Menus, dialogs
- Level 5: 12dp - Modal surfaces

M3 Difference from M2:
- Use tonal surface colors instead of shadows
- Shadows only when required for protection or emphasis
- Components should change elevation on hover (+1 level)

### Motion Physics System

Spring-based animations replacing easing/duration.

Motion Schemes:
- **Expressive**: Overshoots and bounces (recommended for most UIs)
- **Standard**: Minimal bounce, utilitarian

Spring Attributes:
- Stiffness
- Damping
- Initial velocity

Spring Tokens:
| Speed | Use Case |
|-------|----------|
| Fast | Small components (switches, buttons) |
| Default | Partial screen (sheets, rails) |
| Slow | Full-screen animations |

| Type | Behavior |
|------|----------|
| Spatial | Position, size, rotation (overshoots) |
| Effects | Color, opacity (no overshoot) |

Token Format: `md.sys.motion.spring.[speed].[type]`
Example: `md.sys.motion.spring.fast.spatial`

## Components

### Buttons
- **Buttons**: Filled, Outlined, Text, Elevated, Tonal
- **Button Groups**: Related buttons with shape-shifting
- **Split Button**: Button + menu combo
- **Icon Buttons**: Standard, Filled, Outlined, Tonal
- **FAB**: Small, Default, Large, Extended
- **FAB Menu**: FAB with expandable actions
- **Segmented Buttons**: Toggle group
- **Extended FAB**: FAB with label text

### Navigation
- **Navigation Bar**: Bottom nav (3-5 destinations)
- **Navigation Rail**: Side nav for larger screens
- **Navigation Drawer**: Modal or permanent side menu
- **App Bars**: Top/Bottom with actions
- **Tabs**: Primary, Secondary

### Inputs
- **Text Fields**: Filled, Outlined
- **Search**: Persistent, modal
- **Checkbox**, **Radio Button**, **Switch**
- **Sliders**: Continuous, discrete
- **Date Pickers**: Modal, docked, input
- **Time Pickers**: Dial, input

### Selection
- **Chips**: Assist, Filter, Input, Suggestion
- **Menus**: Context, dropdown
- **Lists**: Single/multi-select

### Feedback
- **Dialogs**: Basic, full-screen, modal
- **Snackbar**: Brief messages
- **Progress Indicators**: Linear, circular, waveform
- **Loading Indicator**: Animated wait state
- **Badges**: Notification dots/counts

### Containers
- **Cards**: Elevated, Filled, Outlined
- **Sheets**: Bottom, Side
- **Carousel**: Hero, multi-browse, uncontained
- **Divider**: Horizontal separator

### New M3 Expressive Components
- Toolbars
- Split Button
- Button Groups
- FAB Menu
- Loading Indicator (waveform)

3. Export to platform-specific format

## Glossary (Material A-Z)

### A-B
- **Adaptive design**: Design approach where interface changes based on user, device, or environmental conditions
- **App bar**: Component displaying information and actions at top of screen
- **Banner**: Component displaying prominent message with optional actions
- **Bottom sheet**: Component containing supplementary content anchored to bottom
- **Button**: Component helping people initiate actions (email, share, like)

### C
- **Card**: Component containing content and actions about a subject
- **Checkbox**: Component allowing selection of one or more items from a set
- **Chip**: Component for entering info, making selections, filtering, or triggering actions
- **Color: Baseline scheme**: Default color values for light and dark themes
- **Color: Dynamic color**: User-generated color scheme mapped to app's color scheme
- **Color: Extended color**: Additional colors for custom schemes (brand, semantic)
- **Color: Key color**: Derivation of source color, basis for tonal palette
- **Color: Scheme**: Mapping of color roles to specific tones from a tonal palette
- **Color: Source color**: Single color extracted to define all five key colors
- **Color: Tonal palette**: 13-tone range serving as basis for mapping tones to roles
- **Color: Tone**: Colors with same hue and chroma, varying in lightness
- **Condition**: Signal determining when/how adaptive layout should adapt
- **Contrast**: Difference between colors (40 tone diff = WCAG 3.0, 50 = WCAG 4.5)
- **Customization**: Modification reflecting app, OEM, or user's visual preferences

### D
- **Dark theme**: Low-light UI displaying mostly dark surfaces
- **Data table**: Component displaying data across rows and columns
- **Date picker**: Component for selecting date or date range
- **Design attribute**: Style aspect a token applies to (color, font)
- **Design guidelines**: Descriptive docs demonstrating usage through examples
- **Design specs**: Annotated designs specifying values and parameters
- **Design System Package (DSP)**: Open-format folder structure for sharing design systems
- **Design tokens**: Small, reusable design decisions with self-explanatory names
- **Design tokens: Context**: Conditions where tokens point to non-default values
- **Design tokens: Role**: Shortened version of system token name
- **Design tokens: Types**: Reference (all values), System (theme decisions), Component (element-specific)
- **Design tokens: Value**: Information defining a design attribute
- **Dialog**: Component providing important prompts in user flow
- **Divider**: Thin line grouping content in lists and layouts

### E-H
- **Element**: Part of component a token applies to (container, label text)
- **Extended FAB**: Wider FAB with text label and larger target area
- **FAB (Floating action button)**: Component representing most important action on screen
- **HCT**: Hue, Chroma, Tone - color space enabling dynamic color (based on CAM16)

### I-L
- **Image list**: Component displaying images in organized grid
- **Libraries**: MDC-Android, Jetpack Compose, Flutter, Web
- **List**: Component of continuous, vertical indexes of text or images

### M
- **Material Components**: Open-source UI elements for Android, Flutter, Web
- **Material Design**: Adaptable system of guidelines, components, and tools
  - M1 (2014): First generation, archived
  - M2 (2018): Introduced Material Theming
  - M3 (2021): Includes Material You and dynamic color
- **Material Theming**: Systematic customization to reflect product brand
- **Menu**: Component displaying choices on temporary surface
- **Mode**: Binary system setting (Focus, Airplane, Battery saver)

### N-P
- **Navigation bar**: Persistent way to switch between primary destinations
- **Navigation drawer**: Ergonomic access to destinations
- **Navigation rail**: Access to primary destinations on tablet/desktop
- **Orbiter**: Floating UI elements controlling content in spatial panels
- **Pane**: Building blocks of layout where content is grouped
- **Progress indicator**: Component displaying wait time or process length

### R-S
- **Radio button**: Component for selecting one option from a set
- **Role**: Short nickname describing purpose of design token (On surface, Body 1)
- **Side sheet**: Supplementary content anchored to left or right edge
- **Slider**: Component for selections from range of values
- **Snackbar**: Brief messages about app processes at bottom of screen
- **Spatial**: Placement of UI in space through extended reality (XR)
- **Style**: Customizable properties defining appearance (color, typography, shape)
- **Switch**: Component toggling single item on or off

### T-Z
- **Tab**: Component organizing content across screens and data sets
- **Text field**: Component for entering and editing text
- **Theme**: Set of styles applied globally to adjust appearance
- **Time picker**: Component for selecting and setting specific time
- **Toolbar**: Component displaying navigation and key actions at bottom
- **Tooltip**: Component displaying informative text on hover/focus/tap
- **XR**: Extended reality (VR or passthrough blended with physical environment)
